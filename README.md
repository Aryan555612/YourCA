# YourCA — Cross-Platform Personal Finance Tracker

> **Flutter** (Android + iOS + Web) · **Firebase** (Auth + Firestore + Cloud Functions) · **Riverpod** · **GoRouter** · **fl_chart**

A beautiful, dark-themed personal finance tracking app that works identically on Android, iOS, and Web — with SMS-based auto-import on Android, CSV import on all platforms, smart categorization, and a rule-based savings planner.

---

## Screenshots & Features

| Feature | Description |
|---|---|
| 📱 Phone OTP Auth | Firebase Phone Auth with 6-digit OTP screen |
| 💸 Transaction CRUD | Manual add/edit/delete with category chips |
| 📲 Android SMS Import | Auto-reads bank SMS alerts from 8+ Indian banks |
| 📄 CSV Import | Upload bank statement CSV, map columns, bulk import |
| 🏷️ Smart Categorization | Keyword matching + user correction memory |
| 📊 Dashboard | Monthly income/expense/savings summary + pie + bar charts |
| 🎯 Savings Planner | Spike detection, savings gap, month projection, recurring detection |

## Setup Guide

### Prerequisites

| Tool | Version | Install |
|---|---|---|
| Flutter SDK | 3.32+ | https://docs.flutter.dev/get-started/install/windows |
| Android Studio or VS Code | Latest | For emulators / Xcode for iOS |
| Firebase CLI | Latest | `npm install -g firebase-tools` |
| FlutterFire CLI | Latest | `dart pub global activate flutterfire_cli` |
| Node.js | 20+ | For Cloud Functions |

---

### Step 1 — Clone & Install

```bash
git clone https://github.com/YOUR_USERNAME/YourCA.git
cd YourCA
flutter pub get
```

---

### Step 2 — Create Firebase Project

1. Go to [console.firebase.google.com](https://console.firebase.google.com)
2. **Create project** → name it `yourca-prod`
3. **Build → Authentication → Sign-in methods** → Enable **Phone**
4. **Build → Firestore Database** → Create database (start in test mode, then apply the security rules below)
5. Deploy security rules:
   ```bash
   firebase login
   firebase deploy --only firestore:rules
   firebase deploy --only firestore:indexes
   ```

---

### Step 3 — Connect Firebase to Flutter

```bash
# Activate FlutterFire CLI
dart pub global activate flutterfire_cli

# Configure all three platforms (Android, iOS, Web)
flutterfire configure --project=yourca-prod
```

This generates `lib/firebase_options.dart` and downloads:
- `android/app/google-services.json`
- `ios/Runner/GoogleService-Info.plist`

---

### Step 4 — Run the App

```bash
# Web (in Chrome)
flutter run -d chrome

# Android emulator
flutter run -d emulator-<id>

# iOS simulator (requires macOS + Xcode)
flutter run -d iPhone-<id>
```

---

### Step 5 — Deploy Cloud Functions

```bash
cd functions
npm install
cd ..
firebase deploy --only functions
```

## Android Release APK

```bash
# 1. Generate a signing keystore (one-time)
keytool -genkey -v -keystore yourca-release.jks \
  -alias yourca -keyalg RSA -keysize 2048 -validity 10000

# 2. Configure signing in android/app/build.gradle
#    (add signingConfigs block pointing to yourca-release.jks)

# 3. Build release APK
flutter build apk --release

# Output: build/app/outputs/flutter-apk/app-release.apk

# 4. Distribute
#    Upload to GitHub Releases (https://github.com/YOUR_REPO/releases/new)
#    or Firebase App Distribution:
#    firebase appdistribution:distribute build/app/outputs/flutter-apk/app-release.apk \
#      --app YOUR_ANDROID_APP_ID --groups testers
```

## iOS Ad-Hoc Build

## Web Deployment → Vercel

```bash
# 1. Build Flutter Web
flutter build web --release

# 2. Install Vercel CLI
npm install -g vercel

# 3. Deploy
cd build/web
vercel deploy --prod

# OR: Push to GitHub and connect repo to Vercel
# Set: Build command → "" (pre-built)
#      Output directory → build/web
#      No framework preset
```

The deployed URL will be `https://yourca-XXXX.vercel.app` (Vercel default).

## Android SMS Parser

On Android, YourCA automatically reads incoming bank SMS. Supported banks:

| Bank | Sender IDs |
|---|---|
| SBI | `SBIINB`, `SBIATM`, `SBI-` |
| HDFC | `HDFCBK`, `HDFC-` |
| ICICI | `ICICIB`, `ICICI-` |
| Axis | `AXISBK`, `AXISBN` |
| Kotak | `KOTAKB`, `KOTAK-` |
| PNB | `PNBSMS` |
| IndusInd | `INDBNK` |
| Yes Bank | `YESBNK` |
| IDFC First | `IDFCBK` |

> The app transparently explains why SMS access is needed before requesting permission.

## Savings Suggestion Engine (MVP Rules)

| Rule | Trigger | Action |
|---|---|---|
| Spike detection | Category spend > 20% above 3-month average | Show alert with exact overspend amount |
| Savings gap | Current savings rate < 30% target | Identify top non-essential category to trim |
| Month projection | Linear projection from current pace | "At this rate, you'll save ₹X this month" |
| Recurring detection | Same merchant in 2+ previous months | Flag for review |
